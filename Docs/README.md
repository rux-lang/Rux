# Rux Contributor Docs

Process and architecture documentation for people working **on** the Rux
compiler. If you are looking for how to *use* the language, see the
[website docs](https://rux-lang.dev) instead.

Start with [CONTRIBUTING.md](../CONTRIBUTING.md), then dive into the topic you
need:

- [Development Workflow](Workflow.md) — the day-to-day edit/build/test loop
- [Compiler Architecture](Architecture.md) — components and dependency direction
- [Branch Architecture](Branches.md) — `main`, `dev`, and how branches relate
- [Pull Request Lifecycle](PullRequest.md) — opening, reviewing, and merging PRs
- [CI/CD Flow](CI-CD.md) — what runs on every push and pull request
- [Release Pipeline](Release.md) — turning a tag into a published release

> These are living documents. When you change a process, branch rule, or
> workflow, update the matching page in the **same** pull request.
